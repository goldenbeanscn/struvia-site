# Asset Quality Rules

This document defines quality rules for Struvia engineering evidence images.

The asset system prioritizes engineering credibility over decorative appeal.

---

# Preferred Image Types

Prefer:

* real project photos
* real installation photos
* real detail close-ups
* real factory and QC photos
* real loading and packaging photos
* real site handling photos
* real storage, cutting, cleaning, and protection photos

Accept with caution:

* project-wide facade photos
* factory assembly photos without visible QC action
* logistics proxy images showing movable or modular units but not actual loading
* product detail photos without scale reference

Avoid:

* fake AI buildings
* unrealistic renders
* warped facades
* wrong joints
* impossible trims
* decorative-only images
* heavily blurred or cropped images
* stock-like images without technical relevance

---

# Engineering Quality Checks

Installation evidence:

* worker action or installation stage should be visible
* panel edge, trim, screw, or joint should be identifiable
* image should support a workflow step

Waterproof evidence:

* water path, overlap, drainage exit, flashing, or wet test should be visible
* proxy images must be labeled as proxy

Accessories evidence:

* trim, corner, opening, edge closure, or joint detail should be visible
* decorative facade views are not enough

Product structure evidence:

* panel surface, interlock, cut edge, profile, backing, or layer relationship should be visible

Thermal evidence:

* insulation layer, air gap, wall section, cavity, or transition detail should be visible
* broad exterior photos do not prove thermal behavior

Factory/QC evidence:

* inspection, measuring, packaging, label, surface check, or production process should be visible

Loading/logistics evidence:

* container, pallet, package support, forklift movement, long-panel handling, or loading sequence should be visible

Site handling evidence:

* worker spacing, center support, storage supports, waterproof covering, cutting, cleaning, or debris removal should be visible

---

# Quality Status Labels

Use:

* `usable`: can be used now with normal crop
* `review`: visually useful but requires human approval
* `proxy`: can support a topic but does not prove the specific engineering behavior
* `replace`: unsuitable or misleading
* `needs real image`: no credible image exists yet

---

# Replacement Priority

High:

* waterproof path images
* interlock macro photos
* flashing details
* QC close-ups
* loading/container evidence
* multi-worker handling evidence
* cutting filings removal evidence

Medium:

* better project detail close-ups
* more installation sequence frames
* storage and pallet spacing images
* trim and corner evidence

Low:

* broad project exterior photos
* application card images already supported by existing library
* decorative facade views without visible engineering details

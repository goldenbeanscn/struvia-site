# Evidence Coverage Dashboard

The evidence coverage dashboard tracks whether Technical Center pages have the visual proof needed to support engineering execution.

## Evidence Types

### Real Installation Photos

Purpose:

- Show sequence.
- Show panel seating.
- Show trim closure.
- Show screw zone.
- Show field workflow.

Current coverage:

- Installation sequence evidence exists.
- Interlock and workflow pages use processed installation frames.
- More exact project-specific installation sequences are needed.

### QC Images

Purpose:

- Show factory review.
- Show dimensional or surface control.
- Show assembly condition.
- Support manufacturing credibility.

Current coverage:

- Factory/QC processed images exist.
- More close-up QC inspection records are needed.

### Annotated Diagrams

Purpose:

- Show water path.
- Show thermal continuity.
- Show screw zones.
- Show trim closure.
- Show risk areas.

Current coverage:

- Technical Center has annotation system.
- Strongest pages include annotated evidence.
- Some node diagrams are still schematic-only.

### Project Evidence

Purpose:

- Show real-world execution.
- Validate project type application.
- Connect climate, sequence, inspection, and risk prevention.

Current coverage:

- Project evidence pages exist for five priority project types.
- Some images are representative or proxy evidence.
- Exact project evidence sets are needed.

### Handling Evidence

Purpose:

- Show lifting, storage, transport, site handling, and protection.
- Validate deformation and surface-protection controls.

Current coverage:

- Site handling evidence exists.
- More multi-worker lifting, pallet spacing, and storage protection images are needed.

## Coverage Status Levels

- None: no evidence.
- Placeholder: evidence requirement is described but no image exists.
- Proxy: representative image exists but does not exactly match the detail.
- Embedded: real or representative image is embedded and annotated.
- Verified: exact project or detail image exists and matches the engineering claim.

## Evidence Acceptance Rule

Evidence should be marked verified only when:

- The image shows the specific detail.
- The annotation points to the actual control point.
- The caption explains inspection logic.
- The page identifies what failure the evidence prevents.
- The evidence is linked to a relevant technical system.

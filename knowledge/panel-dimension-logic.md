Reference interpretation:

* It is not a standalone specification sheet.
* Dimensions, loading area, weight, and volume references should be verified against the current order, packaging method, container type, and project requirements.

# Panel Geometry Methodology

Panel geometry should be understood as installation and logistics infrastructure.

Geometry affects:

* facade rhythm
* effective coverage
* accessory layout
* cutting sequence
* package grouping
* container loading
* site handling
* modular coordination

Engineering interpretation:

* panel dimensions do not only describe product size
* they define how wall areas are measured, sequenced, packed, shipped, unloaded, and installed

---

# Effective Width Logic

The PDF identifies the effective width as 38.3 cm.

Engineering meaning:

* effective width is the useful installed coverage width
* it should be used for wall-area estimation and facade rhythm planning
* it supports repeatable alignment because each course or board position follows a predictable coverage module

Workflow use:

* estimate panel quantity by effective coverage, not raw visual width
* coordinate horizontal or vertical rhythm with openings, corners, and trims
* plan waste and cuts around actual installed coverage
* use the same coverage logic for repeated facade bays

Important caution:

* the PDF states that width cannot be customized
* project adaptability should therefore be handled through length planning, cutting, accessory sequencing, and layout design

---

# Installation Rhythm

Effective width creates a repeatable facade module.

Rhythm logic:

* repeated 38.3 cm coverage supports predictable wall-field calculation
* repeated panel width helps maintain alignment consistency
* openings and corners should be coordinated to avoid inefficient narrow cuts
* trim positions should be planned around the repeated width module

Buildability relationship:

* repeatable width reduces layout ambiguity
* predictable coverage supports crew training
* modular rhythm makes inspection easier across larger wall areas

# Thickness & Handling Logic

The PDF identifies 16 mm and 20 mm common thickness options.

Thickness affects:

* handling feel
* panel stiffness behavior
* insulation depth
* package volume
* stack height
* container loading quantity
* trim compatibility

Engineering interpretation:

* 16 mm and 20 mm systems should be evaluated through project needs, climate, handling, packaging, and compatible accessories
* thicker panels may increase volume and weight
* trim and accessory receiving geometry should match the selected panel thickness

Do not infer thermal performance values from thickness alone. Thermal behavior must be evaluated through the complete assembly.

# Lightweight Transport Logic

The PDF gives weight references per 1000 m2:

* 16 mm: 3000 kg
* 20 mm: 3300 kg

Engineering use:

* weight references support early logistics planning
* lower panel mass can simplify handling compared with many heavier facade assemblies
* exact site load and structural demand still require project calculation

Transport implications:

* panel weight affects pallet planning
* weight affects lifting and unloading strategy
* weight must be balanced with volume and container fit
* long panels still require deformation control even when panels are lightweight

# Volume Logic

The PDF gives volume references per 1000 m2:

* 16 mm: 18 m3
* 20 mm: 20 m3

Logistics meaning:

* volume affects container planning, warehouse storage, and package grouping
* thicker panels increase volume per coverage area
* shipment planning must balance area coverage, package dimensions, weight, and container geometry

Warehouse organization:

* group panels by thickness, length, finish, and destination
* keep package labels readable
* avoid mixing lengths that require different loading methods

# Length Compatibility

The PDF states that length can be customized.

Length planning affects:

* wall-area coverage
* cutting waste
* package size
* container compatibility
* site unloading
* installation speed
* modular unit coordination

Length options from PDF container logic:

* 2.9 m
* 3.8 m for 40 ft high-cube logic
* 5.8 m
* other customized lengths when project and transport constraints allow

Engineering caution:

* longer panels can reduce some joints but increase handling and transport risk
* shorter panels can simplify handling but may require more joints or trims
* length selection should consider facade module, container type, worker handling, and unloading conditions

---

# Modular Length Compatibility

Panel length should be coordinated with modular and prefab workflows.

Modular logic:

* align panel length with module height or bay rhythm where possible
* coordinate panel length with transport limits
* plan trim transitions at module joints
* avoid lengths that are efficient for production but difficult to unload or install

Factory-to-site relationship:

* customized length can reduce site cutting
* container-compatible length supports export repeatability
* module-compatible length supports predictable facade sequencing

---

# Site Handling Logic

Dimension choices affect site handling.

Handling considerations:

* long panels require more support points
* wider wall areas require consistent effective-width calculation
* thicker panels may increase stack volume
* panels should not be dragged or twisted during movement
* edge and interlocking geometry must be protected during unloading

Inspection after delivery:

* confirm length and thickness against order
* check package labels
* inspect corners, cut ends, and locking edges
* verify no deformation before installation begins

# Logistics Buildability

Panel dimensions directly affect buildability.

Buildability improves when:

* effective width is used for accurate wall-area planning
* length is selected around installation and transport constraints
* thickness is matched with accessories and handling conditions
* weight and volume are considered before container planning
* loading sequence protects panel geometry

Engineering conclusion:

* a buildable panel system is not only lightweight or customizable
* it is dimensioned, packed, shipped, unloaded, and installed through one coordinated workflow

# Common Transport Risks

## Width Miscalculation

Risk:

* using nominal appearance width instead of effective installed width for quantity planning

Control:

* calculate wall coverage using 38.3 cm effective width unless updated technical documents specify otherwise

## Thickness and Trim Mismatch

Risk:

* selecting trims or accessories that do not match panel thickness

Control:

* confirm 16 mm or 20 mm system before accessory planning

## Long Panel Deformation

Risk:

* 5.8 m or other long panels may bend, twist, or suffer edge damage during loading and unloading

Control:

* use proper support points and avoid dragging or unsupported lifting

## Container Planning Error

Risk:

* choosing panel lengths without confirming 20 ft or 40 ft container compatibility

Control:

* plan length, package grouping, and loading sequence around selected container type

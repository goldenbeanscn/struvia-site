* Dimension references should be used for planning logic and verified per shipment.

# Logistics Methodology

Loading is a protection workflow, not only a space-filling task.

Core logistics logic:

* preserve panel geometry
* preserve carton integrity
* protect interlocking edges
* reduce friction and vibration damage
* use container space efficiently
* support safe unloading
* make the sequence repeatable for export projects

---

# Panel Protection During Transport

Panels must remain installable after transport.

Protection priorities:

* avoid bending long panels
* prevent edge and corner impact
* protect visible coating surfaces
* prevent carton abrasion
* keep packages dry and clean
* avoid loading pressure on interlocking profiles

Workflow controls:

* inspect packaging before loading
* use stable lifting points
* maintain alignment in the container
* prevent package-to-package rubbing where possible
* clean the container before final closure

---

# Container Optimization

Container optimization must balance space efficiency with damage prevention.

Optimization logic:

* match package length to container geometry
* load stable package groups first
* consider forklift movement and turning space
* avoid excessive pressure on lightweight panel stacks
* keep labels accessible where possible
* confirm door closure clearance

Article-derived evidence:

* packaging differs for 2.9 m and 5.8 m panel lengths
* palletized loading and long-panel loading require different handling logic
* container cleanup and door closure are part of the workflow

Dimension PDF evidence:

* 20 ft container compatible lengths: 2.9 m or 5.8 m
* 40 ft high-cube compatible lengths: 2.9 m, 3.8 m, or 5.8 m
* 20 ft loading reference: about 1,500 m2
* 40 ft high-cube loading reference: about 3,500 m2
* these references must be adjusted for packaging method, pallet use, accessories, and order mix

# Dimension-Based Loading Planning

Loading planning should start with panel geometry.

Planning inputs:

* effective width: 38.3 cm
* selected thickness: 16 mm or 20 mm
* selected length: 2.9 m, 3.8 m, 5.8 m, or project-customized length
* container type: 20 ft or 40 ft high-cube
* packaging method: cartons, pallets, film, reinforcement, mixed accessories

Workflow logic:

* calculate area using effective width
* group packages by length and thickness
* confirm volume and weight before container loading
* keep long-panel packages supported during movement
* avoid mixing different length groups without a sequence plan

---

# Weight and Volume Loading Logic

Reference values per 1000 m2 from the dimension PDF:

* 16 mm: 3000 kg and 18 m3
* 20 mm: 3300 kg and 20 m3

Loading interpretation:

* 20 mm panels increase weight and volume compared with 16 mm panels
* higher volume can reduce practical container loading before weight becomes the limit
* pallet and carton dimensions can change actual container utilization
* accessories may consume container space and affect package sequence

Do not treat weight or volume references as automatic loading capacity.

# Pallet Workflow

Pallet workflow supports repeatable export handling.

Primary roles:

* create stable base
* simplify forklift loading
* reduce direct floor contact
* organize packages by order or size
* support receiving-side unloading

Control points:

* pallet support should match package length
* pallet should not create point loading that bends panels
* stacked cartons should remain aligned
* film and reinforcement should be intact before forklift movement

# Loading Efficiency

Loading efficiency should be measured by safe repeatability, not only speed.

Efficient loading means:

* planned sequence
* fewer rehandling steps
* stable package movement
* clear communication between workers
* reduced package adjustment inside container
* predictable unloading order

Risks of poor efficiency:

* rushed handling
* package scraping
* edge collision
* unstable stacking
* carton tearing
* inconsistent loading records

---

# Long Panel Loading Logic

Long panels require stricter loading control.

Risks:

* bending under unsupported spans
* collision with container frame
* twisting during forklift movement
* edge deformation from uneven support
* difficult unloading at destination

Control methods:

* use sufficient support along length
* keep packages level during movement
* avoid dragging
* guide the panel stack into the container slowly
* protect ends from impact
* avoid placing heavy pressure on interlocking edges

---

# Loading Sequence

Recommended sequence:

1. inspect container floor, walls, dryness, and cleanliness
2. confirm package labels and loading plan
3. load base packages or pallets first
4. align packages to reduce movement and friction
5. reinforce or separate vulnerable package contact points
6. load long packages with controlled support
7. inspect remaining clearance and package stability
8. remove loose debris and packaging leftovers
9. close container doors and record loading completion

---

# Export Repeatability

Repeatable loading supports scalable export operations.

Repeatability requires:

* consistent package format
* defined loading method by panel length
* clear QC checklist
* trained handling sequence
* documented container condition
* receiving-side unloading guidance

Buildability relationship:

* export workflow protects the panel geometry needed for site installation
* damaged panels reduce buildability even if manufacturing quality was correct
* logistics must be treated as part of the exterior system, not an afterthought

# Engineering Priority Queue

The engineering priority queue ranks Technical Center work by risk, maturity, and evidence need.

## Priority Criteria

### High-Risk Nodes

Prioritize details where failure has high technical consequence.

Examples:

- Window flashing.
- Base drainage.
- Drainage exit.
- Outside corner.
- Interlocking joint.
- Thermal continuity.
- Cutting corrosion.

### Weak Evidence Pages

Prioritize pages that have strong engineering logic but weak visual proof.

Examples:

- Window flashing.
- Base drainage.
- Thermal continuity joints.
- Outside corner.
- Cutting corrosion node.

### Missing Inspection Systems

Prioritize pages that cannot yet guide verification.

Examples:

- Older node pages.
- Logistics nodes.
- Handling nodes.
- Performance subtopics.

### Low-Maturity Systems

Prioritize pages with weak structure, thin execution logic, or missing failure architecture.

Examples:

- Older node pages.
- Broad category pages without deep subpages.
- Project pages using representative evidence only.

## Priority Levels

### P0: Immediate Engineering Risk

Fix first when a page lacks inspection logic for high-risk conditions.

Examples:

- Blocked drainage.
- Window flashing failure.
- Thermal bridge at joints.
- Unsupported edge.

### P1: Evidence Gap

Fix when the page is technically strong but lacks exact visual evidence.

Examples:

- Replace proxy evidence with real project close-ups.
- Add QC images.
- Add installation sequence frames.

### P2: Depth Improvement

Fix when the page is useful but still overview-heavy.

Examples:

- Add failure chain.
- Add correction workflow.
- Add related node links.

### P3: Maintenance

Fix when page needs consistency, link review, or evidence refresh.

Examples:

- Update related systems.
- Refresh project evidence.
- Review page maturity status.

# Page Maturity Framework

The page maturity framework defines how Technical Center pages progress from initial structure to mature engineering references.

## Maturity Statuses

### Draft

The page exists but may lack complete structure, source alignment, or engineering content.

Minimum condition:

- URL exists.
- Basic page purpose is defined.

### Structured

The page has the required engineering documentation sections.

Minimum condition:

- Technical overview.
- Workflow or sequence.
- Related systems.
- Source or evidence placeholders.

### Engineering Reviewed

The page has been reviewed for technical logic.

Minimum condition:

- Execution logic is clear.
- Sequencing is present.
- Node behavior is explained.
- Failure risks are identified.
- Unsupported claims are avoided.

### Evidence Embedded

The page includes real or representative evidence.

Minimum condition:

- Evidence blocks exist.
- Images or diagrams are annotated.
- Proxy evidence is labeled.
- Missing evidence is identified.

### Inspection Ready

The page can guide field or technical inspection.

Minimum condition:

- Before install checks.
- During install checks.
- After install checks.
- Correction or prevention workflow.
- Evidence review gate.

### Project Connected

The page is connected to real-world project validation.

Minimum condition:

- Related project evidence exists.
- Project page links back to the technical system.
- Climate/application logic is mapped.

### Mature

The page is ready as a reliable engineering reference.

Minimum condition:

- Structured.
- Engineering reviewed.
- Evidence embedded.
- Inspection ready.
- Project connected where relevant.
- Related systems are linked.

## Maturity Scoring

Use a 0-5 score for each dimension:

- Architecture.
- Engineering depth.
- Evidence coverage.
- Inspection readiness.
- Project connection.

Suggested maturity interpretation:

- 0-1: Draft.
- 2: Structured.
- 3: Engineering Reviewed.
- 4: Evidence Embedded or Inspection Ready.
- 5: Mature.

## Current High-Maturity Candidates

- Window flashing.
- Base drainage.
- Interlock sequence.
- Thermal continuity joints.
- Waterproof logic.
- Outside corner.

## Current Common Blockers

- Missing exact project-specific images.
- Weak node pages.
- Incomplete failure chains.
- Lack of project evidence connection.
- Proxy evidence not replaced by real close-ups.

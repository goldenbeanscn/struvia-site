# System Performance Logic

Facade performance is not one number.

It depends on:

* material properties
* composite layer bonding
* panel geometry
* interlocking support
* screw and trim layout
* substrate strength
* drainage and ventilation paths
* fire and code requirements
* workmanship and inspection

Performance hierarchy:

* material level: steel skin, PU core, backing layer, coating, density, conductivity, absorption
* panel level: bonded composite behavior, rigidity, edge closure, dimensional stability
* connection level: interlocking engagement, screw fixing, trim restraint, joint continuity
* facade level: wind load transfer, drainage, thermal continuity, fire strategy, openings, corners
* project level: climate, building height, local code, substrate, maintenance, installation quality

# Engineering Use

Use the spreadsheet to support engineering reasoning:

* thermal data supports insulation and heat-transfer discussion
* wind data supports facade stability and load-path discussion
* moisture data supports water absorption and drainage discussion
* density and bond data support composite material behavior discussion
* heat and fire references support cautious fire-performance interpretation

Do not use the spreadsheet to claim:

* universal code compliance
* certified fire resistance
* guaranteed waterproofing
* guaranteed wind approval for all regions
* complete wall thermal performance without assembly review
* structural capacity independent of fixing and substrate design

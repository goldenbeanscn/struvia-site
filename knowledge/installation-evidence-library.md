# Installation Evidence Library

The installation evidence library defines reusable image types for project validation.

## Evidence Categories

### Sequence Frames

Purpose: show order of work.

Examples:

- Before panel seating.
- During interlock engagement.
- Screw fixing.
- Trim closure.
- Cleaning after cutting.

### Detail Frames

Purpose: show the control point.

Examples:

- Window flashing.
- Base drainage.
- Outside corner trim.
- Interlock joint.
- Screw zone.
- Cut edge.

### Inspection Frames

Purpose: show verification before approval.

Examples:

- Drainage outlet open.
- Head lap visible.
- Jamb edge protected.
- Joint seated.
- Fastener line checked.

### Risk Frames

Purpose: show what needs correction.

Examples:

- Blocked outlet.
- Reverse lap.
- Exposed core.
- Cutting debris.
- Distorted joint.
- Unsupported edge.

### Context Frames

Purpose: show where the detail sits in the project.

Examples:

- Completed wall field.
- Opening in facade context.
- Corner and return plane.
- Warehouse wall run.
- Modular unit exterior.

## Annotation Requirements

Every evidence image should answer:

- What is the control point?
- What sequence step is shown?
- What risk is being prevented?
- What should the inspector check?
- What related system does it validate?

## Current Evidence Assets

Current processed evidence assets include installation frames, product details, project context, factory/QC frames, site handling, logistics, waterproof proxy detail, and accessories/opening trim details.

When a current image is not the exact project or exact detail, label it as representative or proxy evidence and identify the required replacement image.
